# RoboArt.Core – Класс Person

Класс `Person` представляет человека с уникальным идентификатором и проверенными персональными данными. Он налагает строгие правила на имена и автоматически генерирует GUID версии 7.

## Возможности

- **Автоматический идентификатор** – используется `Guid.CreateVersion7()` (требуется .NET 9 или выше).
- **Обязательные поля** – `LastName` и `FirstName` должны быть указаны и проходят валидацию.
- **Необязательное отчество** – `Patronymic` можно опустить или оставить пустым.
- **Валидация** – имена обрезаются от пробелов, проверяются на пустоту и недопустимые символы с помощью пользовательских исключений.

## Члены класса

### `Id` (GUID)
- **Описание:** Уникальный GUID версии 7, назначаемый при создании объекта.
- **Доступ:** только `get`, `init`.

### `LastName` (string)
- **Описание:** Фамилия человека.
- **Валидация:**
    - Не может быть `null`, пустым или состоять только из пробелов.
    - После обрезки должна состоять только из разрешённых символов (определяется `InvalidNameException`).
- **Исключения:**
    - `EmptyNameException` – если входное значение пустое.
    - `InvalidNameException` – если содержит недопустимые символы.

### `FirstName` (string)
- **Описание:** Имя человека.
- **Валидация:** аналогично `LastName`.
- **Исключения:** аналогично `LastName`.

### `Patronymic` (string? – nullable)
- **Описание:** Отчество человека, необязательное.
- **Валидация:**
    - Если `null` или строка из пробелов, свойство остаётся `null`.
    - Если передана непустая строка, она обрезается и валидируется так же, как имя и фамилия.
- **Исключение:** `InvalidNameException` – если указанное (непустое) значение содержит недопустимые символы.

## Пример использования

```csharp
using RoboArt.Core;

try
{
    var person = new Person
    {
        LastName = "Иванов",
        FirstName = "Иван",
        Patronymic = "Иванович"
    };

    Console.WriteLine($"ID: {person.Id}");
    Console.WriteLine($"Имя: {person.LastName} {person.FirstName} {person.Patronymic}");
}
catch (EmptyNameException ex)
{
    Console.WriteLine($"Ошибка имени: {ex.Message}");
}
catch (InvalidNameException ex)
{
    Console.WriteLine($"Недопустимые символы: {ex.Message}");
}
```

## Исключения

Класс использует два пользовательских исключения (предполагается, что они определены в проекте):

- **`EmptyNameException`** – выбрасывается, когда обязательное поле имени получает `null`, пустую строку или строку из пробелов.
- **`InvalidNameException`** – выбрасывается, когда имя содержит символы, запрещённые логикой валидации.

## Требования

- **.NET 9+** – так как используется `Guid.CreateVersion7()`.
- **Пользовательские исключения** – `EmptyNameException` и `InvalidNameException` должны быть доступны в пространстве имён `RoboArt.Core` или в сборке, на которую есть ссылка.

## Примечания

- Все имена перед сохранением обрезаются от начальных и конечных пробелов.
- Сеттер `Patronymic` не выбрасывает исключение для `null` или пустых строк, а просто оставляет свойство равным `null`.
- Свойство `Id` инициализируется один раз при создании объекта и не может быть изменено впоследствии.
